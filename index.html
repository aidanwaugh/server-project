Introduction

Objectives

As a team, create a network in Hyper-V that contains the following:

  • A Windows forest root domain “proj.int”
    • Class C /24 subnet mask (255.255.255.0)
  • A DC server with Active Directory Integrated DNS
  • 1 DHCP server for both the root & child domains
  • Create a Windows child domain “priv.proj.int”
    • Needs 30 host IP addresses > /27 subnet (255.255.255.224)
  • 1 Windows 10 Client VM to manage all computers from the parent domain

1) Create NAT Network Router & Private vSwitches

1.1) Create Internal Switch with a NAT Network

  1. From the host, open PowerShell as an Administrator
  1. Enter the following commands:
    # Create a internal switch named INTNAT, -Verbose is an optional command
New-VMSwitch -SwitchName "INTNAT" -SwitchType Internal -Verbose

# Find the interface index of the virtual switch INTNAT
Get-NetAdapter

# Configure the NAT gateway using New-NetIPAddress
New-NetIPAddress -IPAddress 192.168.3.254 -PrefixLength 24 -InterfaceIndex 42

#Configure the NAT network with the network name and ID/Subnet
New-NetNat -Name INTNAT -InternalIPInterfaceAddressPrefix 192.168.3.0/24

#Connect the INTNAT switch to a adapter for VM DC1
Get-VMSwitch INTNAT | Connect-VMNetworkAdapter -VMName DC1

Server Post-Installation Tasks - Script

  1. Complete the post installation tasks from the new VM with PowerShell with the following:

    ❗ This script is set up for Proj.int 192.168.4.0/ 24 the IP, DNS, DomainName, NetAdapter need to be updated for other networks

    #
# Post-installation tasks
#
$IP = Read-Host -Prompt 'Enter IP 192.168.x.x'
$SrvName = Read-Host -Prompt 'EnterServerNAME'
$SrvOUPath = Read-Host -Prompt 'AD Path OU=X,OU=Servers,DC=proj,DC=int'
$MaskBits = 24
$Gateway = "192.168.4.254"
$Dns = "192.168.4.1"
$IPType = "IPv4"


# Retrieve the network adapter that you want to configure
$adapter = Get-NetAdapter | ? {$_.Status -eq "up"}
# Remove any existing IP, gateway from our ipv4 adapter
If (($adapter | Get-NetIPConfiguration).IPv4Address.IPAddress) {
 $adapter | Remove-NetIPAddress -AddressFamily $IPType -Confirm:$false
}
If (($adapter | Get-NetIPConfiguration).Ipv4DefaultGateway) {
 $adapter | Remove-NetRoute -AddressFamily $IPType -Confirm:$false
}
 # Configure the IP address and default gateway
$adapter | New-NetIPAddress `
 -AddressFamily $IPType `
 -IPAddress $IP `
 -PrefixLength $MaskBits `
 -DefaultGateway $Gateway
# Configure the DNS client server IP addresses
$adapter | Set-DnsClientServerAddress -ServerAddresses $DNS
# Rename the Network Adapter
Rename-NetAdapter -Name "Ethernet" -NewName "ProjLAN"

# Name Computer, add to Domain and OU placement
$cred = Get-Credential proj\aidan
Add-Computer -DomainName proj.int -Credential $cred -OUPath $SrvOUPath
$computer = Get-WmiObject win32_computersystem
$r = $computer.rename("$SrvName", $cred.GetNetworkCredential().Password, $cred.username)

#Rename the local admin account
Rename-LocalUser -Name "Administrator" -NewName "_LSysadmin"
